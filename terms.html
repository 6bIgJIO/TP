    § 7

    Retention and Export

    Local retention

    Incident Packages are stored locally in the configured save directory until deleted by the End User or by an Admin. The Agent does not automatically purge local Incident Packages.

    Vault retention

    The Evidence Vault server supports a configurable RETENTION_DAYS setting. The default is no automatic purging. The Customer is responsible for setting a retention period consistent with applicable legal and regulatory requirements.

    Export

    Individual Incident Packages can be exported from the Evidence Vault as a ZIP archive via the API or dashboard. The ZIP contains the video clip, all metadata files, and the chain-of-custody record.

    § 9

    Security

    • HMAC signing — each Incident Package is signed with a machine-bound HMAC key. Any modification after sealing is detectable.
    • DPAPI key protection — the HMAC key is encrypted using Windows DPAPI, bound to the Windows user account and machine.
    • TPM key protection (Enterprise+) — the HMAC key is protected by the TPM, bound to the physical hardware; cannot be extracted or used on another machine.
    • Hash-linked chain — all incidents are linked via a hash chain; deletion, reordering, or insertion is detectable.
    • Append-only vault — accepted incidents are immutable server-side; no update or delete API exists.
    • TLS transport (optional) — vault upload can be configured for HTTPS with a Customer-provided TLS certificate.

    § 3

    User Cannot Override Administrator Enforcement

    3.1 Technical enforcement

    In Enterprise Mode, settings locked via HKLM are enforced at the application level. The Software will not accept configuration changes to locked keys through the UI, through direct modification of the application configuration file, or through any other mechanism accessible to the End User without administrative privileges.

    • The set-store IPC handler checks isPolicyLocked() before applying any value. Locked keys are rejected.
    • The core process receives its configuration exclusively from the main process at startup. An End User cannot inject alternative configuration without breaking the normal supervised startup path.
    • Windows Event Log entries (Event ID 112/113) record any change to the vault upload state and cannot be suppressed without administrative access to the local Event Log.

    3.2 Limits of enforcement

    The Software enforces Deployment Policy within its own process boundary. It does not and cannot prevent an End User with local administrator privileges from terminating the application, modifying HKCU registry values, deleting locally stored Incident Packages, or blocking network access to the Evidence Vault server.

    Recommendation In high-compliance environments, deploy the Software on locked-down devices where End Users do not have local administrator rights, and lock vault upload via HKLM so that local-only tamper risk is mitigated by the append-only server copy.

    3.3 Audit trail for attempted circumvention

    The chain-of-custody design makes circumvention attempts visible: application termination creates a recording gap; disabling vault upload generates Event ID 113; deleting local Incident Packages creates a sequence number gap detectable by verify.js --chain; clock manipulation at capture time is recorded as a signed timestamp conflict in the Incident Package.
